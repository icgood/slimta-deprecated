slimta.relay.smtp.use_security (self, mode, method, force_verify)
States that SMTP sessions in this relayer should attempt to use TLS security when possible.

Parameters

  • self: the SMTP relayer object.
  • mode: "ssl" for full-connection encryption (i.e. port 465), or "starttls" for encryption as-available.
  • method: a ratchet.ssl context object.
  • force_verify: if true, consider it a complete failure of delivery if the session could not be verified, including certificate validation.
