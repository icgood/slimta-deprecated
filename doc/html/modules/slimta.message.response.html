Module slimta.message.response

Module to represent a response to a message delivery. These responses must come from an SMTP relayer attempting delivery or must be translated into valid SMTP command responses. For example, successfully writing a message into a user's local mailbox should yield a "250" even if an SMTP relayer was not used.

Functions

slimta.message.response.as_http (self)
Returns a response object suitable for returning as an HTTP response. This object includes a code and message, and can also include the arbitrary data string passed to the constructor. SMTP codes are translated (rather ignorantly) into HTTP codes, e.g. 250 => 200.

Parameters

  • self: the message response object.

Return value:

an HTTP response object.

See also:

  • ratchet HTTP server usage
slimta.message.response.as_smtp (self)
Returns the response code and message, as it could be returned to the client of an SMTP edge service.

Parameters

  • self: the message response object.

Return value:

code followed by message.

slimta.message.response.new (code, message, data)
Creates a new message response object.

Parameters

  • code: SMTP code describing the success or failure.
  • message: simple message string describing the response.
  • data: data string providing further information, such as a queue ID. This data is not used by SMTP protocols.

Return value:

a new message response object.
