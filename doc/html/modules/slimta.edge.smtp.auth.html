Module slimta.edge.smtp.auth

Module for managing the authentication procedures of clients connecting to an SMTP edge service. Clients have a choice of any mechanisms the server exposes, such as "PLAIN" or "CRAM-MD5". See RFC 4954.

Functions

slimta.edge.smtp.auth.add_mechanism (self, name, ...)
Adds a new AUTH mechanism to the object, given by name. The extra parameters will vary by mechanism, see the usage manual for supported mechanisms and more information. Note, some mechanisms will only show up to the client if the session has been encrypted.

Parameters

  • self: the auth object.
  • name: the mechanism name, e.g. "PLAIN" or "CRAM-MD5".
  • ...: extra parameters for the mechanism.
