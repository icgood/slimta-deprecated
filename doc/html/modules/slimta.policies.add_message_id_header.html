Module slimta.policies.add_message_id_header

Module that implements policy logic to add a "Message-Id:" header to an outgoing message, as per RFC 2822. This header should be added to all messages that do not already have one.

Functions

slimta.policies.add_message_id_header.add (self, message) If the message does not already contain a "Message-Id:" header, one is constructed and added to the message.
slimta.policies.add_message_id_header.new (hostname, random_func, time_func) Creates a new add_message_id_header object.


Functions

slimta.policies.add_message_id_header.add (self, message)
If the message does not already contain a "Message-Id:" header, one is constructed and added to the message. This header will be of the form "".

Parameters

  • self: the add_message_id_header object.
  • message: the message to add the header to.
slimta.policies.add_message_id_header.new (hostname, random_func, time_func)
Creates a new add_message_id_header object.

Parameters

  • hostname: the hostname of the system generating the message-id. By default, this information will be retrieved manually.
  • random_func: optional function that returns a random string to differentiate this message from others. The default should work just fine.
  • time_func: optional function that returns the current system timestamp. The default is Lua's os.time.

Return value:

a new add_message_id_header object.
