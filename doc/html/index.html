Modules

slimta The root module for all other slimta modules.
slimta.base64 Module containing encoding and decoding functions for the standard BASE64 codec, as provided by the OpenSSL crypto library.
slimta.bus Module that simplifies and abstracts the transfer of objects either locally (within the same Lua state) or over a socket.
slimta.bus.proxy Module that provides a convenient way to apply policies and make changes to a stream of requests traversing a bus.
slimta.edge.http Module that manages an edge listener for HTTP message requests.
slimta.edge.smtp Module that manages an edge listener for SMTP message requests.
slimta.edge.smtp.auth Module for managing the authentication procedures of clients connecting to an SMTP edge service.
slimta.hmac Module for generating HMAC-encoded digests using OpenSSL's HMAC() function.
slimta.message Module for representing email message and all their associated data.
slimta.message.bounce Module for taking a message and response and generating a bounce message to explain to the sender why the original message failed to send.
slimta.message.client Module for describing the client that delivered the message to the edge listener.
slimta.message.contents Module used to parse, store, load, and stringify message contents in an easy, sensible way.
slimta.message.envelope Module for describing the envelope of a message, in other words the info used by policies and the relayer to deliver the message.
slimta.message.response Module to represent a response to a message delivery.
slimta.policies.add_date_header Module that implements policy logic to add a "Date:" header to an outgoing message, as per RFC 2822.
slimta.policies.add_message_id_header Module that implements policy logic to add a "Message-Id:" header to an outgoing message, as per RFC 2822.
slimta.policies.add_received_header Module that implements policy logic to add a "Received:" header to an outgoing message, as per RFC 2821.
slimta.policies.forward Module that implements pattern-based, prioritized recipient rewriting, also known as forwarding.
slimta.queue Module to manage the queue of messages passing through the MTA.
slimta.relay Module that manages the delivery of messages.
slimta.relay.maildrop Module that implements a relayer using the "maildrop" command-line utility.
slimta.relay.smtp Module that attempts message delivery by connection to a remote SMTP server.
slimta.routing.mx Module for standard, RFC-compliant MX-based routing logic.
slimta.routing.static Module for static routing logic for messages.
slimta.signalfd Module for catching signals in an event-based manner.
slimta.storage Module of storage engines currently available to slimta.
slimta.storage.session Module describing the methods exposed by storage engine sessions.
slimta.uuid Module for generating "Universally Unique IDentifiers".
slimta.xml Module that provides an interface to XML parsing using libexpat.
slimta.xml.reader Module that parses XML data into a consistent tree structure that data types may use to recreate objects.
slimta.xml.writer Module which provides a consistent interface for generating XML representations of an object's data, such that it can be recreated without access to the original object.
